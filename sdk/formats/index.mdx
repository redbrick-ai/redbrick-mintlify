<h1 id="formats">Formats<a href="#formats" class="headerlink"
title="Link to this heading">#</a></h1>
<p>This section will document the structure used for importing and
exporting data from RedBrick AI.</p>
<h2 id="import">Import<a href="#import" class="headerlink"
title="Link to this heading">#</a></h2>
<p>The RedBrick AI SDK uses a list of <a
href="#redbrick.types.task.InputTask" class="reference internal"
title="redbrick.types.task.InputTask"><span class="pre"><code
class="sourceCode python">redbrick.types.task.InputTask</code></span></a>
objects for importing data. To import data through the CLI or SDK,
create a JSON file that follows the same format.</p>
<dl>
<dt><em><span class="pre">class</span><span class="w"> </span></em><span
class="sig-prename descclassname"><span
class="pre">redbrick.types.task.</span></span><span
class="sig-name descname"><span class="pre">InputTask</span></span><a
href="../_modules/redbrick/types/task.html#InputTask"
class="reference internal"><span class="viewcode-link"><span
class="pre">[source]</span></span></a><a
href="#redbrick.types.task.InputTask" class="headerlink"
title="Link to this definition">#</a></dt>
<dd>
<p>Represents a single task in RedBrick AI, which is a unit of labeling
work. This user-defined object can contain one or more series of mixed
modalities.</p>
<dl>
<dt><span class="sig-name descname"><span
class="pre">name</span></span><em><span class="p"><span
class="pre">:</span></span><span class="w"> </span><span
class="pre"><code

href="#redbrick.types.task.InputTask.name" class="headerlink"
title="Link to this definition">#</a></dt>
<dd>
<p>A unique user defined string for quickly identifying and searching
tasks.</p>
</dd>
</dl>
