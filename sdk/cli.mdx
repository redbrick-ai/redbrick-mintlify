# Command Line Interface

The RedBrick CLI offers a simple interface to quickly import and export your images & annotations, and perform other high-level actions.

    ```default
    usage: redbrick [-h] [-v] [--completion {bash,zsh,tcsh}] {config,init,clone,info,export,upload,report} ...
    ```

### config

Setup the credentials for your CLI.

    ```default
    redbrick config [-h] [--org ORG] [--key KEY] [--url URL] [--profile PROFILE] [--force] {list,set,add,remove,clear,verify} ...
    ```

### init

Create a new project. We recommend creating a new directory and naming it after your project,
initializing your project within the new directory.

    ```default
    $ mkdir new-project
    $ cd new-project
    $ redbrick init
    ```
    ```default
    redbrick init [-h] [--name NAME] [--taxonomy TAXONOMY] [--reviews REVIEWS] [--workspace WORKSPACE] [--sibling-tasks SIBLING_TASKS] [path]
    ```

### clone

The project will be cloned to a local directory named after your `project name`.

### report

Generate an audit report for a project. Exports a JSON file containing all actions & events
associated with every task, including:

  - Who annotated the task
  - Who uploaded the data
  - Who reviewed the task
  - and more.


    ```default
    redbrick report [-h] [--concurrency CONCURRENCY] [type]
    ```
