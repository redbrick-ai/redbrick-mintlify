<h1 id="command-line-interface">Command Line Interface<a
href="#command-line-interface" class="headerlink"
title="Link to this heading">#</a></h1>
<p>The RedBrick CLI offers a simple interface to quickly import and
export your images &amp; annotations, and perform other high-level
actions.</p>
<div class="highlight-default notranslate">
<div class="highlight">
<pre><code>usage: redbrick [-h] [-v] [--completion {bash,zsh,tcsh}] {config,init,clone,info,export,upload,report} ...</code></pre>
</div>
</div>
<h2 id="named-arguments">Named Arguments<a href="#named-arguments"
class="headerlink" title="Link to this heading">#</a></h2>
<dl>
<dt><kbd>-v, --version</kbd></dt>
<dd>
<p>show program’s version number and exit</p>
</dd>
<dt><kbd>--completion</kbd></dt>
<dd>
<p>Possible choices: bash, zsh, tcsh</p>
<p>print shell completion script</p>
</dd>
</dl>
<h2 id="commands">Commands<a href="#commands" class="headerlink"
title="Link to this heading">#</a></h2>
<dl>
<dt><kbd>command</kbd></dt>
<dd>
<p>Possible choices: config, init, clone, info, export, upload,
report</p>
</dd>
</dl>
<section id="Sub-commands">
<h2 id="sub-commands">Sub-commands<a href="#Sub-commands"
class="headerlink" title="Link to this heading">#</a></h2>
<h3 id="config">config<a href="#config" class="headerlink"
title="Link to this heading">#</a></h3>
<p>Setup the credentials for your CLI.</p>
<div class="highlight-default notranslate">
<div class="highlight">
<pre><code>redbrick config [-h] [--org ORG] [--key KEY] [--url URL] [--profile PROFILE] [--force] {list,set,add,remove,clear,verify} ...</code></pre>
</div>
</div>
<h4 id="positional-arguments">Positional Arguments<a
href="#positional-arguments" class="headerlink"
title="Link to this heading">#</a></h4>
<dl>
<dt><kbd>sub_command</kbd></dt>
<dd>
<p>Possible choices: list, set, add, remove, clear, verify</p>
</dd>
</dl>
<section id="named-arguments_repeat1">
<h4 id="named-arguments-1">Named Arguments<a
href="#named-arguments_repeat1" class="headerlink"
title="Link to this heading">#</a></h4>
<dl>
<dt><kbd>--org, -o</kbd></dt>
<dd>
<p>Org ID</p>
</dd>
<dt><kbd>--key, -k</kbd></dt>
<dd>
<p>API Key</p>
</dd>
<dt><kbd>--url, -u</kbd></dt>
<dd>
<p>Endpoint URL, should default to https://api.redbrickai.com.</p>
</dd>
<dt><kbd>--profile, -p</kbd></dt>
<dd>
<p>Profile name</p>
</dd>
<dt><kbd>--force, -f</kbd></dt>
<dd>
<p>Force create new credentials</p>
<p>Default: False</p>
</dd>
</dl>
</section>

<section id="named-arguments_repeat5">
<h4 id="named-arguments-5">Named Arguments<a
href="#named-arguments_repeat5" class="headerlink"
title="Link to this heading">#</a></h4>
<dl>
<dt><kbd>--with-files</kbd></dt>
<dd>
<p>Export with files (e.g. images/video frames)</p>
<p>Default: False</p>
</dd>
<dt><kbd>--dicom-to-nifti</kbd></dt>
<dd>
<p>Convert DICOM images to NIfTI. Applicable when <span
class="pre"><code
class="code docutils literal notranslate">--with-files</code></span> is
set.</p>
<p>Default: False</p>
</dd>
<dt><kbd>--old-format</kbd></dt>
<dd>
<p>Whether to export tasks in old format. (Default: False)</p>
<p>Default: False</p>
</dd>
<dt><kbd>--without-masks</kbd></dt>
<dd>
<p>Exports only tasks JSON without downloading any segmentation masks.
Note: This is not recommended for tasks with overlapping labels.</p>
<p>Default: False</p>
</dd>
<dt><kbd>--semantic</kbd></dt>
<dd>
<p>Whether to export all segmentations as semantic_mask. This will
create one segmentation file per class. If this is set to True and a
task has multiple instances per class, then attributes belonging to each
instance will not be exported.</p>
<p>Default: False</p>
</dd>
<dt><kbd>--binary-mask</kbd></dt>
<dd>
<p>Whether to export all segmentations as binary masks. This will create
one segmentation file per instance.</p>
<p>Default: False</p>
</dd>
<dt><kbd>--single-mask</kbd></dt>
<dd>
<p>Whether to export all segmentations in a single file. Binary mask
will be considered if both binary_mask and single_mask are set.</p>
<p>Default: False</p>
</dd>
<dt><kbd>--no-consensus</kbd></dt>
<dd>
<p>Whether to export tasks without consensus info. If None, will default
to export with consensus info, if it is enabled for the given
project.</p>
<p>Default: False</p>
</dd>
<dt><kbd>--png</kbd></dt>
<dd>
<p>Export labels as PNG masks</p>
<p>Default: False</p>
</dd>
<dt><kbd>--rt-struct</kbd></dt>
<dd>
<p>Export labels as DICOM RT-Struct. (Only for DICOM images)</p>
<p>Default: False</p>
</dd>
<dt><kbd>--mhd</kbd></dt>
<dd>
<p>Export segmentation masks in MHD format.</p>
<p>Default: False</p>
</dd>
<dt><kbd>--clear-cache</kbd></dt>
<dd>
<p>Clear local cache</p>
<p>Default: False</p>
</dd>
<dt><kbd>--concurrency, -c</kbd></dt>
<dd>
<p>Concurrency value (Default: 10)</p>
<p>Default: 10</p>
</dd>
<dt><kbd>--stage, -s</kbd></dt>
<dd>
<p>Export tasks that are currently in the given stage. Applicable only
with <span class="pre"><code
class="code docutils literal notranslate">redbrick</code></span><code
class="code docutils literal notranslate"> </code><span
class="pre"><code
class="code docutils literal notranslate">export</code></span> and <span
class="pre"><code
class="code docutils literal notranslate">redbrick</code></span><code
class="code docutils literal notranslate"> </code><span
class="pre"><code
class="code docutils literal notranslate">export</code></span><code
class="code docutils literal notranslate"> </code><span
class="pre"><code
class="code docutils literal notranslate">latest</code></span></p>
</dd>
<dt><kbd>--destination, -d</kbd></dt>
<dd>
<p>Destination directory (Default: current directory)</p>
<p>Default: “.”</p>
</dd>
</dl>
</section>

<section id="named-arguments_repeat6">
<h4 id="named-arguments-6">Named Arguments<a
href="#named-arguments_repeat6" class="headerlink"
title="Link to this heading">#</a></h4>
<dl>
<dt><kbd>--as-frames</kbd></dt>
<dd>
<p>Upload video from image frames</p>
<p>Default: False</p>
</dd>
<dt><kbd>--type, -t</kbd></dt>
<dd>
<p>Possible choices: DICOM3D, NIFTI3D, NRRD3D, IMAGE2D, VIDEO,
VIDEOFRAMES</p>
<p>Import file type [’<span class="pre"><code
class="code docutils literal notranslate">DICOM3D</code></span>’, ‘<span
class="pre"><code
class="code docutils literal notranslate">NIFTI3D</code></span>’, ‘<span
class="pre"><code
class="code docutils literal notranslate">NRRD3D</code></span>’, ‘<span
class="pre"><code
class="code docutils literal notranslate">IMAGE2D</code></span>’, ‘<span
class="pre"><code
class="code docutils literal notranslate">VIDEO</code></span>’, ‘<span
class="pre"><code
class="code docutils literal notranslate">VIDEOFRAMES</code></span>’]</p>
<p>Please refer to <a
href="https://docs.redbrickai.com/importing-data/direct-data-upload"
class="reference external">our documentation</a>, to understand the
required folder structure and supported file types.</p>
<p>Default: “DICOM3D”</p>
</dd>
<dt><kbd>--as-study</kbd></dt>
<dd>
<p>Group files by study</p>
<p>Default: False</p>
</dd>
<dt><kbd>--json</kbd></dt>
<dd>
<p>Upload json files with list of task objects</p>
<p>Default: False</p>
</dd>
<dt><kbd>--segment-map, -m</kbd></dt>
<dd>
<p>Segmentation mapping file path</p>
</dd>
<dt><kbd>--storage, -s</kbd></dt>
<dd>
<p>Storage method: (redbrick [default], public, )</p>
<p>Default: “redbrick”</p>
</dd>
<dt><kbd>--label-storage</kbd></dt>
<dd>
<p>Label Storage method: (same as items storage <span class="pre"><code
class="code docutils literal notranslate">--storage</code></span>
[default], redbrick, public, )</p>
</dd>
<dt><kbd>--ground-truth</kbd></dt>
<dd>
<p>Upload tasks directly to ground truth.</p>
<p>Default: False</p>
</dd>
<dt><kbd>--label-validate</kbd></dt>
<dd>
<p>Validate NIfTI label instances and segmentMap. By default, the
uploaded NIfTI files are not validated during upload, which can result
in invalid files being uploaded. Using this argument validates the files
before upload, but may increase the upload time.</p>
<p>Default: False</p>
</dd>
<dt><kbd>--prune-segmentations</kbd></dt>
<dd>
<p>Prune segmentations that are not part of the series.</p>
<p>Default: False</p>
</dd>
<dt><kbd>--rt-struct</kbd></dt>
<dd>
<p>Upload segmentations from DICOM RT-Struct files.</p>
<p>Default: False</p>
</dd>
<dt><kbd>--mhd</kbd></dt>
<dd>
<p>Upload segmentations from MHD files.</p>
<p>Default: False</p>
</dd>
<dt><kbd>--clear-cache</kbd></dt>
<dd>
<p>Clear local cache</p>
<p>Default: False</p>
</dd>
<dt><kbd>--concurrency, -c</kbd></dt>
<dd>
<p>Concurrency value (Default: 10)</p>
<p>Default: 10</p>
</dd>
</dl>
